# 17 · 事件统计前端信息架构与体验重设计

> 状态：✅ P0 已落地  
> 对应后端：`13-event-statistics-engine.md`（不变）  
> 目标：拆分「发起回测 / 任务列表 / 结果解读」，降低单页密度；图表可读优先。

---

## 1. 问题诊断（现状）

当前 `/event-stats` 单页堆叠了：

1. 发起表单（策略、区间、宇宙 4 种模式、观测窗、三并发）
2. 运行中进度条（取消 / 配置摘要）
3. 历史任务全量表格（配置列过多 + 重跑/删除）
4. 摘要卡片（1/3/5/10 日收益）
5. 粗糙 SVG 多 Tab 图表
6. 三张宽统计表（收益 / 路径 / 时间）
7. 事件明细大表

**后果**：首屏不知从哪下手；列表无限变长；结果区「表 + 图」信息重复且图无教学语境。

---

## 2. 设计原则

| # | 原则 |
|---|------|
| 1 | **一页一事**：发起 / 列表 / 详情分离 |
| 2 | **详情才重**：聚合解读 + 图表 + 明细只在详情页 |
| 3 | **列表可扫**：只留识别任务所需的最小列 |
| 4 | **图表先讲结论**：每个图配一句人话 + 图例，不堆裸柱 |
| 5 | **进度全局可见但不抢戏**：列表页顶条 / 详情页侧条均可，避免双份完整表单 |
| 6 | **URL 可分享**：`/event-stats`、`/event-stats/runs/:runId` |

---

## 3. 路由与页面拆分

```text
/event-stats                    → EventStatsHomePage（发起 + 任务列表）
/event-stats/runs/:runId        → EventStatsRunDetailPage（结果详情）
```

导航文案保持「事件统计」；从簇页跳入仍可用 query：  
`/event-stats?universe=cluster&profile=…&cluster=…`（仅预填发起表单）。

兼容：旧链接 `?run=xxx` → 重定向到 `/event-stats/runs/xxx`。

---

## 4. 页面 A：任务台（Home）

### 4.1 布局（自上而下）

```text
┌─────────────────────────────────────────────┐
│ 页头：事件统计 · 一句话说明                   │
├─────────────────────────────────────────────┤
│ [可选折叠] 发起统计  ▾ / ▴                   │
│   默认：折叠；点「新建统计」展开              │
│   运行中：顶部 sticky 进度条（取消+摘要）     │
├─────────────────────────────────────────────┤
│ 任务列表                                     │
│   筛选：状态 | 策略                          │
│   表格（精简列）+ 分页                       │
└─────────────────────────────────────────────┘
```

### 4.2 发起区（折叠默认收起）

分组，减少一行挤满：

| 分组 | 字段 |
|------|------|
| 基础 | 策略、观测窗、日期快捷 + 起止 |
| 宇宙 | 模式切换；按模式显示二级控件（自选 / 单簇 / 簇抽样） |
| 性能 | 日/匹配/Observe 并发（默认收在「高级」折叠内） |

主按钮唯一：**开始统计**。运行中禁用并显示进度条。

### 4.3 任务列表（精简 + 分页）

**列（V1）**

| 列 | 内容 |
|----|------|
| 时间 | `created_at` 相对/绝对 |
| 策略 | pattern + version |
| 区间 | start → end |
| 宇宙 | 一行摘要（全市场 / 自选 N / 簇抽样 N） |
| 结果 | 事件数；SUCCESS 时附 5 日均收益（小字） |
| 状态 | badge + 运行中进度% |
| 操作 | **查看** · 重跑 · 删除 |

去掉：观测窗独立列（放进查看详情）、过长宇宙原文。

**分页**

- 每页默认 **10**，可选 10 / 20 / 50  
- 后端：`GET /api/event-stats/runs?limit=&offset=` 已有 limit，**补 offset + total**  
- 前端：页码器（上一页 / 下一页 / 第 n 页）

**行交互**

- 单击行 → 进入详情页（不再在本页展开报告）  
- 运行中任务：行内显示细进度；顶栏同步进度条可取消

## 5. 页面 B：任务详情（Run Detail）

路由：`/event-stats/runs/:runId`

### 5.1 顶栏

```text
← 返回列表    任务 de3a5972…    SUCCESS
策略 RANGE_BREAKOUT · 观测窗 20 日 · 宇宙摘要
[重跑] [删除]     （运行中则 [取消]）
```

配置详情用 **「查看完整配置」** 抽屉/折叠，默认不占首屏。

### 5.2 结论区（先读懂）

3～4 张结论卡 + 一句自动生成摘要，例如：

> 共 154 次命中，覆盖 69 只股票。5 日均收益 +1.9%，胜率 49%；10 日均收益 +1.7%。P10～P90（5 日）约 -5.6%～+12.6%，波动偏大。

卡片建议：

1. 覆盖（事件数 / 股票数 / 样本质量 ok·truncated）  
2. 短线（1+3 日：均收益·胜率）  
3. 中线（5+10 日）  
4. 风险（平均 MAE / 最大回撤）

### 5.3 图表区（重设计，可读优先）

**弃用** 当前多 Tab 裸柱「自己猜」。改为固定 3 块、每块有标题+图例+解读：

#### 图 1 · 远期收益概览（主图）

- 类型：**分组柱状**（每期一个组：均收益）+ **折线叠加胜率**（右轴 0–100%）  
- X：1日 / 3日 / 5日 / 10日 / 20日 / 60日 / 窗末  
- 正收益绿色、负收益红色；胜率灰色折线  
- 脚注：`均收益 = 所有事件该 horizon 收益的算术平均；胜率 = 收益>0 占比`

#### 图 2 · 收益离散度（P10–中位–P90）

- 类型：**水平区间条**（箱须简化版）  
- 每行一个 horizon：粗线 P10→P90，圆点中位数，竖线 0%  
- 脚注：`中间 80% 的事件落在色带内；色带越长，结果越不稳定`

#### 图 3 · 10 日收益分布

- 类型：**直方图**（事件级，最多 500 条）  
- 标注均值竖线、0 轴  
- 脚注：`看分布是偏单边还是两边肥尾`

路径风险（MFE/MAE/回撤）放入小号第四卡或折叠「路径图」，默认不抢主视线。

**实现建议（V1）**

- 继续零依赖 SVG，但统一组件：`StatCallout` / `GroupedReturnChart` / `SpreadStripChart` / `HistogramChart`  
- 交互：hover 显示精确值；不强制引入 recharts（体积与风格一致性优先）  
- 若后续要刷子缩放再引入轻量库

### 5.4 数据表区（折叠）

默认折叠两档：

1. **聚合表**（远期收益 / 路径 / 时间）— 给要核对数字的人  
2. **事件明细** — 分页（每页 20），列：代码、信号日、相似度、1/3/5/10 日、MFE、MAE、质量  

明细支持按 10 日收益排序（沿用 API）。

### 5.5 运行中详情

若 `status=RUNNING`：

- 大进度条 + message  
- 配置摘要  
- 取消按钮  
- 不展示空图表/空表（避免「全是 —」）

## 6. API 小改（配合分页）

```http
GET /api/event-stats/runs?limit=10&offset=0&status=&pattern_id=
→ { total, limit, offset, runs: [...] }
```

- `total`：必加  
- `offset`：必加  
- `status` / `pattern_id`：可选过滤（P0 可先做前端过滤，P1 后端）

详情接口不变；事件列表沿用 `limit/offset`。

## 8. 视觉与文案

- 沿用现有 CSS 变量（`--accent` 绿 / 负值红），**不**新开紫色/奶油风  
- 详情页图表区浅底分隔，避免「又一张 data table」感  
- 所有专业词悬停 `title` 或脚下小字：P10、P90、MFE、MAE、胜率  
- 空态：无任务 → CTA「新建统计」；无事件 → 说明可能被冷却去重清空
