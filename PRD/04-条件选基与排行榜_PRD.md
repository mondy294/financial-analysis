# PRD 04：条件选基与排行榜

> 优先级：P1
> 
> 目标：让项目从“已知基金查询工具”升级为“基金发现工具”。

---

## 1. 背景

当前 `financial` 的使用方式是：
- 先知道基金代码
- 再去查询详情

这意味着产品在“发现候选基金”这个环节是空白的。用户如果不知道买什么、看什么，就无法进入主流程。

竞品启发：
- 天天基金：条件选基、大数据榜、股票选基、基金对比
- 蚂蚁理财金选：统一分析指标、多赛道筛选、透明评分

因此，需要建设一个**本地版、可解释、非黑盒**的选基中心。

---

## 2. 目标

建设一个“条件选基与排行榜”模块，支持：
1. 按条件筛选基金
2. 按主题/类型/风险收益排序
3. 按板块/主题浏览基金池
3. 提供透明评分卡
4. 将结果加入自选、对比、持有流程

---

## 3. 非目标

本阶段不做：
- 完全覆盖全市场所有基金的所有字段
- 神秘算法推荐
- 实时高频排行榜
- 复杂投顾组合推荐

重点是先做一个**本地可用、数据口径清晰、可解释**的候选池工具。

---

## 4. 关键设计原则

### 4.1 先做“可解释筛选”，不是“猜你想买什么”
必须告诉用户：
- 筛选条件是什么
- 评分维度是什么
- 排序为什么会这样

### 4.2 先做缓存型基金池，不做现抓全市场
当前架构是本地工作台，不适合每次打开页面都去现抓海量基金。
应采用：
- 定时拉取/手动刷新市场基金池缓存
- 本地筛选和排序

### 4.3 先做 V1 的可闭环基金池
允许 V1 先覆盖：
- 常见主动基金
- 指数基金
- 纯债基金
- 固收+基金

不要求第一版就极度完整。

---

## 5. 用户场景

### 场景 A：找同类中更稳的基金
用户诉求：
- 想找红利/医疗/宽基/纯债里的更优候选
- 关心收益、回撤、波动、费用

### 场景 B：为已有持仓找补充品
用户诉求：
- 现有组合偏科技，想找低相关或偏稳健品种补充

### 场景 C：每周扫一遍市场候选池
用户诉求：
- 快速查看近期强势、低回撤、低费率、规模适中的基金列表

### 场景 D：先选板块，再看板块内基金
用户诉求：
- 不想先知道具体基金代码
- 想先看“红利 / 医疗 / 科技 / 新能源 / 消费 / 宽基 / 债基”等板块
- 点进某个板块后，直接看到该板块下的基金列表，再继续筛选、排序、加入自选

---

## 6. 页面与入口设计

### 路由
新增：`/screener`

### 页面结构
1. 顶部筛选器区
2. 板块导航区 / 板块快捷入口
3. 排行榜快捷 tab
4. 结果列表
5. 单只基金评分抽屉/侧栏
6. 快捷动作区（加自选 / 加对比 / 看详情）

### 板块浏览模式
页面应支持两种进入方式：
1. 先选筛选条件，再看结果
2. 先选板块，再看该板块下基金列表

### 板块入口建议
可在页面顶部或左侧增加“热门板块”入口，例如：
- 红利
- 医疗
- 科技
- 新能源
- 消费
- 宽基
- 纯债
- 固收+

点击任一板块后：
- 自动带出该板块标签
- 展示该板块基金列表
- 允许继续叠加收益、回撤、波动、费率等筛选条件
- 页面上清楚展示“当前板块”

### 首页快捷入口建议
可在左侧导航加入：
- 条件选基
- 排行榜

## 7. 功能拆解

## 7.1 筛选器
### 筛选维度（V1）
- 基金类型：主动、指数、纯债、固收+
- 板块/主题：红利、医疗、科技、新能源、消费、宽基、债基、海外、黄金等
- 主题/行业：红利、医疗、科技、新能源、消费、宽基等
- 区间收益下限：1 月 / 3 月 / 6 月 / 1 年
- 最大回撤上限
- 波动率上限
- 费率上限
- 规模范围
- 成立年限范围
- 是否可定投

### 高级筛选（V2）
- 基金经理任职年限
- 同类排名分位
- 风格标签
- 重仓行业暴露

### 板块筛选说明
- “板块/主题”应作为一级浏览入口，不只是一个普通筛选项
- 用户可以只选一个板块直接看列表，也可以在板块下继续叠加其它条件
- 一个基金允许归属多个板块标签
- 若基金没有明确板块标签，可归入“其他”或“不完整数据”

---

## 7.2 排行榜
### 默认榜单建议
- 近 1 月收益榜
- 近 3 月收益榜
- 近 1 年收益榜
- 低回撤榜
- 低波动榜
- 高性价比榜
- 稳健底仓榜
- 高弹性进攻榜

### 板块榜单
支持在板块维度下查看榜单，例如：
- 红利板块收益榜
- 医疗板块低回撤榜
- 科技板块高弹性榜
- 宽基板块稳健底仓榜

榜单页应支持：
- 先选板块，再选榜单类型
- 或在榜单结果中切换板块

### 说明
“稳健底仓榜”“高弹性进攻榜”属于规则榜单：
- 稳健底仓榜：低波动 + 低回撤 + 中等以上收益 + 费率友好
- 高弹性进攻榜：高收益 + 高弹性 + 行业/主题鲜明

必须在榜单页展示规则说明。

---

## 7.3 透明评分卡
### 评分维度建议（100 分）
- 收益能力：30
- 稳定性：20
- 回撤控制：20
- 费用友好度：10
- 管理持续性：10
- 规模/成立时长健康度：10

### 展示要求
- 结果列表上显示总分
- 点击后展开分项得分与解释
- 支持按“总分 / 某单项分”排序

### 示例解释
- “收益强，但波动偏大，适合高风险偏好用户。”
- “费率与回撤表现更好，更适合作为长期底仓候选。”

---

## 7.4 保存筛选方案
### 功能目标
允许用户保存常用筛选器，例如：
- “低波红利底仓”
- “医疗观察池”
- “纯债防守候选”

### 数据保存
新增本地文件：
- `data/screener-presets.json`

---

## 7.5 与现有流程打通
### 每条结果支持动作
- 查看详情
- 加入自选
- 加入对比
- 录入持有

### 目标
把筛选页变成“候选池入口”，不是终点页。

### 板块页目标
把“板块”做成发现基金的第一层入口：
- 用户先看赛道/板块
- 再看板块内基金列表
- 最后进入详情、自选、对比、持有流程

## 8. 数据层设计

## 8.1 新增本地缓存文件
- `data/fund-universe-cache.json`

### 建议结构
```json
{
  "updatedAt": "2026-04-15T10:00:00.000Z",
  "items": [
    {
      "code": "270042",
      "name": "某基金",
      "fundType": "混合型",
      "sectorTags": ["医疗", "创新药"],
      "themeTags": ["医疗"],
      "metrics": {
        "return1m": 3.2,
        "return3m": 8.1,
        "return1y": 14.6,
        "maxDrawdown1y": -18.2,
        "volatility1y": 22.5,
        "feeRate": 1.2,
        "size": 38.5,
        "establishedYears": 5.6
      },
      "score": {
        "total": 76,
        "return": 24,
        "stability": 15,
        "drawdown": 12,
        "fee": 9,
        "management": 8,
        "health": 8
      }
    }
  ]
}
```

### 字段补充说明
- `sectorTags`：用于板块浏览与板块榜单，例如“红利 / 医疗 / 科技 / 新能源 / 消费 / 宽基 / 纯债 / 固收+”
- `themeTags`：用于更细粒度的主题标签，例如“创新药 / AI / 半导体 / 中证红利 / 沪深300”
- 一个基金可同时命中多个 `sectorTags` 和多个 `themeTags`
- V1 可先采用本地规则映射或手工维护标签，不要求一开始做到完全自动化

## 8.2 缓存刷新策略
### V1
- 手动刷新按钮
- 后端按需拉取并更新缓存
- 缓存过期时间：24 小时

### V2
- 启动时检查缓存时效
- 可接入本地定时任务

## 9. API 设计

### 新增接口建议
- `GET /api/screener/options` —— 获取筛选选项
- `POST /api/screener/query` —— 按条件查询基金池
- `GET /api/screener/sectors` —— 获取可浏览板块列表
- `GET /api/screener/sectors/:sector/funds` —— 获取某板块下基金列表
- `POST /api/screener/refresh` —— 刷新本地基金池缓存
- `GET /api/screener/presets`
- `POST /api/screener/presets`
- `DELETE /api/screener/presets/:id`

### 查询请求体示例
```json
{
  "fundTypes": ["主动", "指数"],
  "sectors": ["红利"],
  "themes": ["红利"],
  "minReturn1y": 10,
  "maxDrawdown1y": 18,
  "maxFeeRate": 1.2,
  "sortBy": "score.total",
  "sortOrder": "desc"
}
```

## 11. 错误态与边界处理

### 数据不足
- 若市场池缓存为空：提示刷新基金池
- 若某基金缺少部分字段：允许展示，但标记“数据不完整”
- 若某板块下暂无基金：显示空状态，并允许返回全部基金池

### 性能控制
- 单次结果默认显示前 100 条
- 支持分页或“加载更多”

### 合规文案
- 显示“本页面结果仅为条件筛选与研究辅助，不构成投资建议”

---

## 12. 验收标准

### 功能验收
- 可按多条件筛选基金
- 可按板块浏览基金池，并展示板块下基金列表
- 可生成多个榜单
- 每只基金都能看到透明评分卡
- 可将筛选结果一键送入自选、对比、详情流程

### 数据验收
- 本地缓存可刷新
- 缓存过期与空缓存时有合理提示

### 体验验收
- 3 分钟内可以完成一次条件选基
- 1 分钟内可以完成一次“选择板块 -> 浏览基金 -> 查看详情”的操作
- 排序与筛选结果变化清晰、可解释
