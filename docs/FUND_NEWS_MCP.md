# 基金市场新闻 MCP 说明

## 这次新增了什么

新增了一个 MCP 工具：`get_fund_market_news`

目标很直接：

> 按时间段查询近期国内外、可能影响基金判断的市场新闻，而不是只盯基金净值和持仓。

这次先接的是 **东方财富全球财经快讯公开接口**，已经验证可直接拉数据，不是空文档占坑。

---

## 数据来源

当前实现基于东方财富公开快讯接口：

- 基础域名：`https://np-weblist.eastmoney.com/`
- 实际接口：`/comm/web/getFastNewsList`

项目里实际使用的是这些栏目聚合：

- 焦点
- 基金
- 全球股市
- 商品
- 外汇
- 债券
- 地区（中国 / 海外）
- 央行（中国 / 海外）
- 经济数据（中国 / 海外）

这意味着它不只是“新闻列表”，而是偏向：

- 市场焦点
- 央行 / 利率 / 汇率
- 海外股市扰动
- 宏观数据发布
- 基金行业相关快讯

更适合基金场景。

---

## MCP 工具名

`get_fund_market_news`

### 输入参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `startTime` | `string` | 是 | 开始时间，支持 `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss` |
| `endTime` | `string` | 是 | 结束时间，支持 `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss` |
| `regions` | `string[]` | 否 | 可选 `国内` / `海外` |
| `topics` | `string[]` | 否 | 可选 `焦点` / `基金` / `全球股市` / `商品` / `外汇` / `债券` / `地区` / `央行` / `经济数据` |
| `keywords` | `string[]` | 否 | 关键词过滤，例如 `美联储`、`人民币`、`黄金`、`AI` |
| `limit` | `number` | 否 | 最多返回多少条，默认 200，最大 500 |

---

## 返回结构

返回结果里重点看这些字段：

- `coverageNote`
  - 当前新闻源范围说明
- `feedStats`
  - 每个栏目实际抓了多少页、命中了多少条、是否触发截断
- `stats.byTopic`
  - 按主题统计条数
- `stats.byRegion`
  - 按地区统计条数
- `items`
  - 真正的新闻列表

### 单条新闻字段

| 字段 | 说明 |
| --- | --- |
| `title` | 标题 |
| `summary` | 快讯正文 / 摘要 |
| `publishedAt` | 发布时间 |
| `detailUrl` | 东方财富详情链接（如果接口带 code） |
| `topic` | 主题栏目 |
| `region` | `国内` / `海外` / `综合` |
| `importanceScore` | 影响优先级分数 |
| `impactTags` | 自动推断的影响标签，如 `货币政策` / `宏观数据` / `权益市场` / `汇率` |
| `relatedStocks` | 接口里带出的相关股票线索 |

## 为什么这个方案适合当前项目

因为现在的基金 Agent 之前更像：

- 看净值
- 看均线
- 看持仓股
- 看同类对标

但解释“最近一周为什么这样”“未来 1-3 个月可能被什么扰动”时，外部证据偏弱。

这个 MCP 补上的就是：

- 美联储 / 中国央行动向
- CPI / PMI / 非农 / GDP 这类宏观数据
- 海外股市和汇率波动
- 债券收益率变化
- 商品价格扰动
- 基金行业直接相关新闻

简单说，就是让 Agent 别再只盯图，开始看风向。

## 当前边界

### 1. 这是公开快讯源，不是正式授权数据库
优点：
- 不额外配 token
- 接得快
- 覆盖范围够广

边界：
- 更适合近期滚动事件
- 不保证超长历史范围稳定
- 页面接口未来可能改版

### 2. 当前更适合近 31 天
为了避免公开源分页过深导致不稳定，当前实现限制在 **近 31 天** 内。

### 3. 当前是“快讯聚合”，不是深度长文库
它适合：
- 找近期催化
- 找外部扰动
- 给 Agent 补证据链

它不适合直接替代：
- 深度研报库
- 全量长文新闻数据库
- 舆情情绪建模系统

## 后续可扩展方向

如果后面要继续做深，可以往这几个方向扩：

1. **接入 Tushare 的 `news` / `major_news` / `eco_cal`**
   - 换更深历史
   - 补长篇新闻和财经日历
   - 但需要 token / 权限

2. **把新闻 MCP 接进现有基金 Agent 的自动推理链**
   - 用户问“最近一周为什么跌”时自动补新闻证据

3. **做基金 / 板块 / 关键词级别的新闻归因摘要**
   - 例如自动归纳：`人民币走强 + 美债回落 + 半导体回暖`

4. **给前端加“最近影响因素”卡片**
   - 直接在基金详情页展示最近几条高相关快讯

---

## 结论

这次新增的不是“又一个能查数据的小工具”，而是给项目补上了一块之前很缺的外部信息层：

> 让基金分析不只看净值和持仓，也能把最近真正扰动市场的新闻、央行动作和宏观数据拉进来。
